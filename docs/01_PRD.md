# READi ERP
# Product Requirements Document (PRD)

Version : 1.0

---

# 1. 프로젝트 목적

READi ERP는 Readi Robust Machine의 구매, 재고 및 운영 데이터를 하나의 시스템에서 관리하기 위한 ERP 프로젝트이다.

초기에는 Airtable을 Database로 사용하며,
향후 SQL Database로 전환 가능하도록 설계한다.

본 프로젝트의 목표는

"완벽한 ERP를 만드는 것"이 아니라

"회사에서 실제 사용할 수 있는 ERP를 점진적으로 구축하는 것"

이다.

---

# 2. 개발 철학

## 핵심 기능 우선

처음부터 모든 기능을 만들지 않는다.

현재 업무에 가장 필요한 기능부터 개발한다.

충분히 검증한 후 다음 기능을 추가한다.

---

## 확장 가능한 구조

기능은 언제든 추가할 수 있도록 설계한다.

하지만 아직 필요하지 않은 기능 때문에
현재 구조를 복잡하게 만들지 않는다.

---

## 단순함 유지

가능하면 가장 단순한 구조를 선택한다.

복잡한 설계보다

쉽게 이해되고
쉽게 유지보수 가능한 구조를 우선한다.
