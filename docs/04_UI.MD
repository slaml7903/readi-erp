# READi ERP
# UI Structure

Version : 1.0

---

# 목적

이 문서는 READi ERP의 기본 화면 구조와 메뉴 구성을 정의한다.

초기 목표는 화려한 디자인이 아니라
업무자가 빠르게 조회하고 입력할 수 있는 화면을 만드는 것이다.

---

# UI 개발 원칙

## 1. 단순함 우선

처음부터 복잡한 화면을 만들지 않는다.

핵심 기능을 빠르게 사용할 수 있도록 구성한다.

---

## 2. 모듈형 메뉴

각 업무는 독립된 메뉴로 구분한다.

- Dashboard
- 구매관리
- 재고관리
- BOM
- 차량관리
- 관리자

---

## 3. 모바일 대응 고려

초기 개발은 PC 기준으로 진행한다.

단, 현장 작업자가 사용할 가능성이 있으므로
모바일에서도 깨지지 않도록 구성한다.

---

# 기본 레이아웃

READi ERP는 아래 구조를 기본으로 한다.

┌────────────────────────────────────────────┐
│ Header │

├──────────────┬─────────────────────────────┤
│ Sidebar │ Main Content │
│ │ │
│ Dashboard │ 선택한 메뉴 화면 │
│ 구매관리 │ │
│ 재고관리 │ │
│ BOM │ │
│ 차량관리 │ │
│ 관리자 │ │
└──────────────┴─────────────────────────────┘

---

# Header

상단 영역.

## 표시 항목

- READi ERP
- 현재 접속 사용자
- 알림
- 설정

초기 Version에서는 사용자명과 알림은 표시만 하고,
실제 기능은 나중에 구현한다.

---

# Sidebar

좌측 메뉴.

## 메뉴 구성

### Dashboard

- 전체 현황

### 구매관리

- 구매요청
- 발주
- 입고확인
- 거래처

### 재고관리

- Item Master
- 재고현황
- 입고등록
- 출고등록

### BOM

- BOM 조회
- 제품별 구성품
- 생산가능수량

### 차량관리

- 차량현황
- 운행일지
- 예약현황

### 관리자

- 시스템 설정
- 사용자 관리
- 기준정보

---

# Dashboard 화면

첫 화면.

## 목적

ERP 전체 현황을 한눈에 보여준다.

## 표시 항목

- 구매 진행 건수
- 입고 예정 건수
- 재고 부족 품목
- 차량 예약 현황
- 최근 업데이트

---

# 구매관리 화면

## 목적

기존 Airtable 구매 DB를 웹에서 조회하고 관리한다.

## 초기 기능

- 구매요청 목록 조회
- 발주 목록 조회
- 입고확인 목록 조회

## 향후 기능

- 구매요청 등록
- 발주 생성
- 입고 처리
- 지출 상태 관리

---

# 재고관리 화면

## 목적

재고 현황과 입출고를 관리한다.

## 초기 기능

- Item Master 조회
- 현재고 조회
- 입고 등록
- 출고 등록

---

# BOM 화면

## 목적

완제품, 반제품, 하위 부품 관계를 관리한다.

## 초기 기능

- BOM 목록 조회
- 상위품목 기준 하위품목 조회
- 하위품목 기준 사용 제품 조회

---

# 차량관리 화면

## 목적

법인차량 운행 및 예약 정보를 관리한다.

## 초기 기능

- 차량현황 조회
- 운행일지 조회
- 예약현황 조회

---

# 관리자 화면

## 목적

시스템 기준정보와 설정을 관리한다.

초기 Version에서는 화면만 구성하고
실제 기능은 나중에 구현한다.

---

# 화면 개발 순서

## Step 1

공통 레이아웃

- Header
- Sidebar
- Main Content

## Step 2

Dashboard

## Step 3

구매관리

## Step 4

재고관리

## Step 5

BOM

## Step 6

차량관리

---

# 디자인 방향

- 단순함
- 업무용
- 빠른 조회
- 카드형 Dashboard
- 테이블 중심 화면
- 모바일 대응 가능 구조

---

# 완료 기준

Version 1 UI는

사용자가 메뉴를 이해하고
필요한 화면으로 이동할 수 있으면 완료로 본다.

처음부터 완벽한 디자인을 목표로 하지 않는다.
